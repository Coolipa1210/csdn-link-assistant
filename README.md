# CSDN Link Assistant

CSDN 好友文章链接整理助手，用于从 CSDN 页面和白名单私信对话中收集文章链接，按状态整理、去重、导入导出，并辅助维护白名单待办流程。

所有数据默认保存在浏览器本地。本扩展不读取、不保存 CSDN 密码、Cookie 或 Token，也不是 CSDN 官方工具。使用时请遵守 CSDN 平台规则，并优先使用人工确认流程。

## 主要功能

### 获取文章链接

- 扫描当前 CSDN 页面里的文章链接，自动去重。
- 按白名单分组扫描最近 1-30 天私信对话中的文章链接。
- 配置“我的作者 ID”，自动忽略自己的文章，避免误加入待处理列表。
- 支持手动添加、批量导入、复制 JS 数组或 JSON。
- 支持按状态管理链接：待处理、已导出、已忽略。
- 支持一键清空已导出链接或删除全部链接记录。

### 人员分组

- 以白名单分组管理用户，默认提供“默认分组”。
- 支持导入用户 ID、私信链接或 CSDN 主页链接。
- 支持 TXT 导入和导出，一行一个用户 ID。
- 支持从当前 CSDN 私信页面一键提取用户 ID 并加入白名单。
- 支持启用、停用、删除用户，并按分组展开管理。
- 支持为用户获取并缓存私信 iframe 地址，后续处理时优先复用。

### 自动私信处理

- 选择白名单分组和固定文本后，先展示发送预览。
- 默认建议使用审核模式：只填入内容，不自动发送，由用户手动确认。
- 关闭审核模式后，可按队列逐个处理白名单用户，并尝试点击发送按钮。
- 处理过程支持暂停、继续、失败重试。
- 系统会复用当前标签页，优先切换已缓存的私信 iframe，必要时再使用普通私信地址兜底。
- 日志页会记录每个用户的处理结果，方便回看成功、失败和重试情况。

## 使用流程

1. 在白名单页导入或新建分组，维护需要关注的用户。
2. 在链接页扫描当前页面，或选择白名单分组扫描私信对话中的文章链接。
3. 在链接列表中筛选、忽略、导出待处理文章链接。
4. 在私信页选择分组和固定文本，使用审核模式或队列处理模式完成白名单私信待办。

## 安装

1. 打开 Chrome 或 Edge 的扩展管理页面。
2. 开启“开发者模式”。
3. 选择“加载已解压的扩展程序”。
4. 选择本项目目录。

## 体验细节

- 可点击“持久侧边栏”打开不会被网页点击关闭的面板。
- 插件会记住上次打开的功能页，下次打开自动回到同一页。
- 本地记录任务日志，不保存账号凭证。

## 打包

发布浏览器扩展时，将以下文件放在 zip 根目录：

- `manifest.json`
- `popup.html`
- `popup.css`
- `popup.js`
- `background.js`
- `content.js`
- `core.js`
- `assets/`
